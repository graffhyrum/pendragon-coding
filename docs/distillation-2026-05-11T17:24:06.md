## Patterns Worth Repeating

- **Pattern**: Block-and-direct hooks
  **Condition**: When writing PreToolUse or PostToolUse guard hooks
  **Summary**: Exit 2 with a remediation command in the message rather than silently auto-mutating state. Preserves operator audit trail.

- **Pattern**: Dependency boundary patching
  **Condition**: When encountering third-party TS type conflicts
  **Summary**: Use patchedDependencies instead of app-level type casts. Keeps fixes localized, verifiable, and portable.

- **Pattern**: Extraction paired with value-pinning tests
  **Condition**: When extracting magic values/inline literals to named constants
  **Summary**: Commit the value assertion in the same change as the extraction. The constant name documents intent; the test pins the value.

- **Pattern**: Repo-first discovery
  **Condition**: When a request mixes codebase discovery with persistence
  **Summary**: Inspect the repo first, ask the user only for gaps you cannot infer from code/docs. Saves user effort.
